## ProductCard design decisions (all applied)

### 1. Brand name above product title
Brand renders above the product name in `font-body text-small uppercase tracking-wide text-text-muted` — no "By " prefix. New optional `brand` prop; `description` made optional (omitted on browse listing where no per-product copy exists from the DB).

### 2. Clickable card + hover-reveal Quick Add
- `onAddToCart` replaced with `onCardClick` (full-card hit target, cursor pointer, keyboard accessible via `role="button"` + Enter key) and `onQuickAdd` (ghost `+` button).
- Quick Add is `absolute bottom-md right-md`, `opacity-0` at rest, `group-hover:opacity-100` on card hover, `focus-visible:opacity-100` for keyboard. Stops propagation so it doesn't trigger navigation.
- Sizing: `rounded-full border border-border bg-surface px-3 py-xs text-small text-text-muted`. (`px-3` is Tailwind's 12px default — between `sm`/8px and `md`/16px tokens.)
- Ghost hover: `hover:border-accent hover:text-accent`.

### 3. Ghost Button variant added to the system
New `ghost` variant documented in `docs/design.md` (§4 Components) and `docs/component-spec.md` (§6 Button): `border-border text-text-muted` at rest, `border-accent text-accent` on hover. For low-priority inline actions where primary/secondary would compete with content hierarchy.

## Bug found and fixed: category filter was silently broken

**Root cause:** Products are assigned to **subcategories** (e.g. "All-Purpose Cleaner", "Hand Soap"), not to the top-level categories shown in the filter ("Cleaning", "Personal Care", "Pantry"). Filtering by a top-level `category_id` returned zero results — which fell through to displaying all products unchanged.

**Fix:** When a top-level category is selected, `BrowsePage` collects all child category IDs from the already-loaded `categories` state and passes the full list (`[parentId, ...childIds]`) to `getProducts({ categoryIds })`. `getProducts` now uses `.in('category_id', ids)` for multi-ID queries.
