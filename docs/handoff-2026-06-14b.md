## Key decisions and discoveries

### 1. `products` has no `slug` column
The initial schema uses `id` (UUID) as the product identifier. `slug` exists only on `categories`, `brands`, and `certifications`. **The detail route will be `/product/:id`**, not `/product/:slug`. This is already captured in the build-plan starting-state note so Session 5 doesn't re-derive it.

### 2. score ‚Üí safety mapping (locked)
```
75‚Äì100  ‚Üí  "clean"
40‚Äì74   ‚Üí  "caution"
0‚Äì39    ‚Üí  "avoid"
```
Implemented as `scoreToSafety(score)` in `src/lib/catalog.ts`. Applied to every `ProductSummary` and `ProductDetail` returned by the catalog.

## Catalog module (`src/lib/catalog.ts`) ‚Äî exported API

| Export | Description |
|---|---|
| `SafetyLevel` | `"clean" \| "caution" \| "avoid"` |
| `Category` | `{ id, slug, name, parentId, sortOrder }` |
| `ProductSummary` | `{ id, name, brand, categoryId, categoryName, imageUrl, cleanScore, safety }` |
| `ProductDetail` | `ProductSummary + { barcode, retailerUrl, ingredients[], certifications[] }` |
| `Ingredient` | `{ name, concernLevel, concernReason, position }` |
| `Certification` | `{ slug, name }` |
| `scoreToSafety(score)` | Maps 0‚Äì100 integer ‚Üí `SafetyLevel` |
| `getCategories()` | All categories ordered by `sort_order` |
| `getProducts({ categoryId? })` | All products or filtered by category |
| `getProductById(id)` | Full product detail with ingredients + certifications; returns `null` if not found |
| `searchProducts(query)` | `ilike` search on product name |

**Note:** `searchProducts` currently only matches on `products.name`. Searching by brand name requires a join and is scoped to Session 4 (where search is the feature being built).
