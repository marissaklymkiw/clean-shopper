## Key decisions made this session

- **Save ≠ Cart.** Saving is a bookmark (interest, research); cart is intent to buy. Same `cart_items` table, differentiated by `item_type`.
- **Single cart page, not tabs.** Cart at top, "Saved for later" below — the Target/Amazon pattern. More scannable and avoids hidden state behind a tab.
- **`user_id` must be explicit in inserts.** RLS policy is `auth.uid() = user_id` — Supabase does not auto-inject it. `getCurrentUserId()` fetches it from `supabase.auth.getUser()` before every insert.
- **Optimistic updates.** Context state updates immediately on toggle; DB call follows. If the DB call fails, the UI is momentarily inconsistent — acceptable for V1.
